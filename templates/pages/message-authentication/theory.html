Theory

Introduction to Message Authentication

Message authentication is a mechanism or service used to verify that a message has not been modified while in transit (integrity) and that the purported source of the message is authentic (authenticity).

Message Authentication Codes (MACs)

A Message Authentication Code (MAC) is a short piece of information used to authenticate a message. A MAC algorithm accepts as input a secret key and an arbitrary-length message to be authenticated, and outputs a MAC value.

The MAC value protects both a message's data integrity and its authenticity by allowing verifiers to detect any changes to the message content.

Hash-based Message Authentication Codes (HMACs)

HMAC is a specific type of message authentication code involving a cryptographic hash function and a secret cryptographic key. It may be used to simultaneously verify both the data integrity and authenticity of a message.

HMAC(K,m) = H((K' ⊕ opad) || H((K' ⊕ ipad) || m))

Where:
H = cryptographic hash function
K = secret key
K' = derived key
|| = concatenation
⊕ = XOR
opad = outer padding
ipad = inner padding

Digital Signatures

Digital signatures are a mathematical scheme for verifying the authenticity of digital messages or documents. A valid digital signature gives the recipient reason to believe that the message was created by a known sender and that it wasn't altered in transit.

Security Properties

  • Authenticity: Verifies the origin of the message
  • Integrity: Ensures the message hasn't been modified
  • Non-repudiation: Sender cannot deny sending the message (in case of digital signatures)
