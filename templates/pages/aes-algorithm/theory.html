Advanced Encryption Standard (AES) - Theory

Introduction

The Advanced Encryption Standard (AES) is a symmetric block cipher algorithm established by the U.S. National Institute of Standards and Technology (NIST) in 2001. It processes data blocks of 128 bits using cipher keys of 128, 192, or 256 bits. AES replaced the Data Encryption Standard (DES) as the approved standard for a wide range of applications.

Algorithm Structure

  • 128-bit block size organized as a 4×4 matrix of bytes
  • Key sizes: 128 bits (10 rounds), 192 bits (12 rounds), or 256 bits (14 rounds)
  • Each round consists of four transformations (except the final round)
  • State array undergoes transformations in each round

Round Transformations

1. SubBytes Transformation

A non-linear substitution where each byte is replaced with another according to a lookup table (S-box)

  • Provides confusion
  • Uses multiplicative inverse in GF(2⁸)
  • Followed by an affine transformation

2. ShiftRows Transformation

A cyclic shifting operation on the rows of the state:

  • Row 0: no shift
  • Row 1: 1-byte shift left
  • Row 2: 2-byte shift left
  • Row 3: 3-byte shift left

3. MixColumns Transformation

A linear mixing operation operating on columns:

  • Each column is treated as a polynomial
  • Multiplied modulo x⁴ + 1 with a fixed polynomial
  • Provides diffusion in the cipher

4. AddRoundKey Transformation

Key addition operation:

  • Bitwise XOR with the round key
  • Round keys derived from the cipher key
  • Different round key for each round

Key Expansion

The AES key expansion algorithm generates a series of Round Keys from the Cipher Key:

  • Takes the cipher key and produces a key schedule
  • Uses key expansion function to generate round keys
  • Each round key is 128 bits
  • Number of round keys depends on key size

Security Features

  • Strong resistance against known attacks
  • No practical attacks that break AES
  • Mathematical properties ensure security:
    • Non-linearity of S-box
    • Diffusion through MixColumns
    • Key mixing in each round

Implementation Considerations

  • Hardware and software implementations possible
  • Optimization techniques available:
    • T-table approach
    • Bit-slicing
    • Hardware acceleration
  • Side-channel attack protections needed
