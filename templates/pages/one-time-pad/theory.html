One-Time Pad (OTP) Theory

What is a One-Time Pad?

A One-Time Pad (OTP) is an encryption technique that uses a random key of the same length as the plaintext, where each key is used exactly once. When implemented correctly, it provides perfect secrecy, making it theoretically impossible to break even with unlimited computational power.

Key Characteristics

  • The key must be truly random
  • The key must be at least as long as the message
  • Each key must be used only once
  • The key must be kept completely secret
  • The key must be securely shared between parties

Mathematical Foundation

The One-Time Pad uses modular addition for encryption and subtraction for decryption:

Encryption: Ci = (Pi + Ki) mod 26
Decryption: Pi = (Ci - Ki) mod 26

where Pi is the plaintext letter, Ki is the key letter, and Ci is the ciphertext letter (all converted to numbers 0-25).

Perfect Secrecy

The One-Time Pad achieves perfect secrecy because:

  • Every possible plaintext is equally likely given any ciphertext
  • The key is truly random and as long as the message
  • Each key is used only once
  • The ciphertext provides no information about the plaintext

Practical Limitations

Despite its perfect security, OTP has practical challenges:

  • Key generation must be truly random
  • Key distribution is challenging
  • Key management for large messages is difficult
  • Keys must be securely stored and never reused
