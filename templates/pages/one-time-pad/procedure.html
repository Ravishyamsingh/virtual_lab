One-Time Pad (OTP) Cipher Procedure

Encryption Process

  1. Generate the One-Time Pad (Key):
    • Create a truly random sequence of numbers
    • The key must be the same length as the plaintext
    • Each key must be used only once
  2. Encryption Steps:
    • Convert plaintext letters to numbers (A=0, B=1, ..., Z=25)
    • Add each plaintext number to corresponding key number
    • Take modulo 26 of the sum
    • Convert result back to letters

Decryption Process

  1. Using the Same Key:
    • Convert ciphertext letters to numbers
    • Subtract the key numbers from ciphertext numbers
    • Take modulo 26 of the difference
    • Convert result back to letters

Important Guidelines

  • The key must be truly random
  • The key must be at least as long as the message
  • Each key must be used exactly once
  • Both sender and receiver must securely store the key
  • The key must be securely transmitted and kept secret

Example Workflow

1. Plaintext: HELLO

2. Convert to numbers: 7 4 11 11 14

3. Random key: 15 9 2 4 19

4. Add and mod 26:

(7+15)%26=22 (4+9)%26=13 (11+2)%26=13 (11+4)%26=15 (14+19)%26=7

5. Convert back to letters:

W N N P H

The ciphertext is: WNNPH

Mathematical Formula

Encryption:

Ci = (Pi + Ki) mod 26

Decryption:

Pi = (Ci - Ki) mod 26

Where:

  • Ci = Ciphertext letter position
  • Pi = Plaintext letter position
  • Ki = Key letter position
