Theory: Cryptographic Hash Functions

What is a Hash Function?

A cryptographic hash function is a mathematical algorithm that maps data of arbitrary size to a fixed-size output (hash value). It's designed to be a one-way function, making it computationally infeasible to reverse the process.

Key Properties

1. Deterministic

The same input will always produce the same hash value.

2. Quick Computation

It should be fast to calculate the hash value for any given input.

3. Pre-image Resistance

It should be computationally infeasible to find an input that hashes to a specific output.

4. Collision Resistance

It should be computationally infeasible to find two different inputs that hash to the same output.

Common Hash Functions

MD5 (Message Digest Algorithm 5)

- 128-bit hash value
- No longer considered cryptographically secure
- Still used for checksums and data integrity verification

SHA-1 (Secure Hash Algorithm 1)

- 160-bit hash value
- Deprecated for security applications
- Practical collision attacks have been demonstrated

SHA-256

- 256-bit hash value
- Part of the SHA-2 family
- Widely used in modern security applications
- No known practical attacks

The Avalanche Effect

The avalanche effect is a desirable property of cryptographic hash functions where if an input is changed slightly, the output hash changes significantly. For example, changing a single bit in the input should change approximately half of the bits in the output hash.

Applications

Password Storage

Storing hashed passwords instead of plaintext for security.

Digital Signatures

Creating message digests for digital signatures.

Blockchain

Creating unique block identifiers and maintaining chain integrity.

File Integrity

Verifying file integrity through hash comparison.
