DSA Algorithm Implementation Assignment

Assignment Overview

This assignment focuses on implementing and understanding the Digital Signature Algorithm (DSA) through hands-on exercises and theoretical analysis.

Task 1: Basic DSA Implementation (25 points)

Objective: Implement a basic DSA key generation and signature system.

  • Generate DSA parameters (p, q, g) with appropriate sizes
  • Create a key pair (private key x, public key y)
  • Sign a given message using your implementation
  • Verify the signature using the public key

Deliverables: Code implementation with comments explaining each step

Task 2: Security Analysis (20 points)

Objective: Analyze the security properties of DSA implementation.

  • Explain the role of the discrete logarithm problem in DSA security
  • Discuss the importance of proper random number generation
  • Analyze what happens if the same k value is used twice
  • Compare DSA security with RSA and ECDSA

Deliverables: 2-3 page security analysis report

Task 3: Parameter Analysis (15 points)

Objective: Understand the relationship between DSA parameters.

  • Experiment with different parameter sizes
  • Measure the impact of parameter size on performance
  • Verify that q divides (p-1) for your generated parameters
  • Test the generator g properties

Deliverables: Parameter analysis with performance measurements

Task 4: Signature Verification Deep Dive (20 points)

Objective: Understand the mathematical foundation of signature verification.

  • Manually verify a signature using the verification formula
  • Explain why the verification equation v = r works
  • Demonstrate what happens with an invalid signature
  • Show the step-by-step verification process

Deliverables: Mathematical proof and verification examples

Task 5: Practical Implementation (20 points)

Objective: Create a practical DSA application.

  • Build a simple document signing application
  • Implement proper error handling and input validation
  • Add support for different hash functions
  • Create a user-friendly interface

Deliverables: Working application with user documentation

Theory Questions

Question 1: Mathematical Foundation (5 points)

Explain the discrete logarithm problem and its role in DSA security. Why is it computationally difficult to solve?

Question 2: Parameter Selection (5 points)

Why must q divide (p-1) in DSA? What would happen if this condition is not met?

Question 3: Random Number Importance (5 points)

What are the security implications if the same random number k is used to sign two different messages?

Question 4: Signature Verification (5 points)

Walk through the mathematical steps that prove why the verification equation v = r works in DSA.

Question 5: Comparison Analysis (5 points)

Compare DSA with RSA signatures in terms of signature size, computation speed, and security assumptions.

Practical Exercises

Exercise 1: Parameter Generation

Generate DSA parameters with the following specifications:

  • q: 160-bit prime
  • p: 1024-bit prime where q divides (p-1)
  • g: generator of order q modulo p

Document the generation process and verify the parameters.

Exercise 2: Signature Analysis

Sign the message "Hello, DSA!" using your implementation and:

  • Show all intermediate values
  • Verify the signature manually
  • Modify one bit of the message and show verification failure
Exercise 3: Performance Testing

Measure the performance of your DSA implementation:

  • Key generation time
  • Signature generation time
  • Signature verification time
  • Compare with different parameter sizes

Submission Guidelines

  • Format: Submit all code, reports, and documentation in a single ZIP file
  • Code: Include well-commented source code with clear variable names
  • Documentation: Provide README file explaining how to run your implementation
  • Reports: Submit reports in PDF format with proper formatting
  • Testing: Include test cases demonstrating your implementation works correctly
